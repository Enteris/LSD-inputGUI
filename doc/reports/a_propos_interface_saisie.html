Développement de l'interface de saisie du simulateur



  • Contexte:

L'interface graphique du simulateur est la partie usager du  nouveau simulateur du laboratoire de simulation et de dépistage LSD. Elle contient un assistant de saisie de données et un programme d'affichage des résultats.

Le développement de cette interface utilisateur se fait en Python sous Eclipse, en utilisant XML/DOM pour format d'échange de données et Qt comme bibliothèque graphique.

  • Étapes majeures:

  1. Conception: Étape déja finie. Définission des besoins, des attentes, des contraintes. Élaboration de la solution et du plan d'action. Choix d'architecture et d'outils de développement.

  2. Implantation du Loader et du Saver: Étape en cours. Codage des différentes fonctions qui chargent les données à partir d'un fichier XML et ceux qui enregistrent les données sous format XML reconnue par le moteur du simulateur. Tests de validation.

  3. Implantation des vues: Étape planifiée. Design et codage de toutes les fenêtres de l'interface graphique

  4. Integration des Modules: Étape planifiée. Implantation des liens entre les différents modules cités ci-haut. Tests d'integration, tests de d'acceptation.

  • Python, XML, Qt... C'est quoi?

    • Python: un langage de programmation révolutionnaire open-source, à la fois orienté-objet, script, fonctionnel... avec une syntaxe très parlante et assez rapide lors de l'exécution. C'est pour ça qu'on l'a choisi pour ce projet. On utilise PyDev, le module qui permet de développer en Python sous Eclipse.

    • Eclipse: un environnement de développement integré (en anglais, IDE). C'est une platforme de développement informatique open-source où vous pouvez développer des applications pour des projets d'envergure. Que ça soit en Java ou en C++, en Agile ou en RUP, grâce à la facilité d'y rajouter des plugins spécifiques pour chaque outil, Eclipse est l'environnement de développement par excellence. Il existe des plugins pour travailler en équipe à distance (SVN et CVS), pour faire des beaux diagrammes, pour faire le suivi de votre projet (Gantt)... Et pour compiler de programmes écrits dans votre langage de programmation préféré.

    • XML: un méta-langage extensible pour encapsuler les données sous un format texte lisible à la fois par un humain que par une machine, et ceci suivant une arborescence que vous définissez. N'importe quelle sorte de données peut être enregistrée sous format XML selon votre hiérarchie. Les nouvelles versions de fichiers HTML (pages Web) ne sont qu'une forme standard de XML entre bien d'autres. XML est utilisé dans notre projet pour la communication entre l'utilisateur et le simulateur, l'interface de saisie vise à rendre cet échange de données le plus simple qui soit.

    • Qt: une bibliothèque qui permet de faire des programmes avec une interface graphique conviviale (en anglais GUI). Sous une distribution libre et initialement destinnée pour C++, Qt resteune propriété du géant des télécoms suédois Nokia. Elle fournit une librairie de classes utiles pour créer des fenêtres, des tableaux, des menus... Et même des boîtes de dessin. PyQt, une réimplantation de Qt pour Python, est utilisée dans le cadre de ce projet pour créer les vues du programme.
